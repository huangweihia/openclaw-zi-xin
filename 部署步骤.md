## 一、创建新容器（推荐）

### 方案 A：新建 Docker 容器（推荐）

```bash
# 1. 在 Docker Desktop 创建新容器
# 容器名称：ai-side-laravel-max
# 端口映射：8082:80
# 挂载卷：/home/node/.openclaw/workspace/ai-side-laravel-max:/var/www/html

# 2. 或在 WSL 中执行
docker run -d \
  --name ai-side-laravel-max \
  -p 8082:80 \
  -v /home/node/.openclaw/workspace/ai-side-laravel-max:/var/www/html \
  -e APACHE_DOCUMENTROOT=/var/www/html/public \
  laravelsail/php82-composer:latest

# 3. 进入容器
docker exec -it ai-side-laravel-max bash

# 4. 安装依赖
composer install --no-dev --optimize-autoloader

# 5. 配置环境
cp .env.example .env
php artisan key:generate

# 6. 配置数据库（编辑 .env）
DB_CONNECTION=mysql
DB_HOST=host.docker.internal
DB_PORT=3306
DB_DATABASE=ai-side-laravel-max
DB_USERNAME=root
DB_PASSWORD=your_password

# 7. 运行迁移
php artisan migrate --force

# 8. 设置权限
chmod -R 775 storage/ bootstrap/cache/

# 9. 优化缓存
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

## 七、故障排查

### 问题 1：500 错误

```bash
# 查看日志
tail -f storage/logs/laravel.log

# 清除缓存
php artisan cache:clear
php artisan config:clear
php artisan route:clear
php artisan view:clear

# 重新缓存
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

### 问题 2：数据库连接失败

```bash
# 检查 .env 配置
cat .env | grep DB_

# 测试数据库连接
php artisan tinker
>>> DB::connection()->getPdo();
```

### 问题 3：权限错误

```bash
# 检查权限
ls -la storage/
ls -la bootstrap/cache/

# 修复权限
chmod -R 775 storage/ bootstrap/cache/
chown -R www-data:www-data storage/ bootstrap/cache/
```

### 问题 4：端口冲突

```bash
# 检查端口占用
netstat -tulpn | grep 8082

# 修改端口（编辑 docker-compose.yml 或 Docker Desktop 端口映射）
# 将 8082 改为其他端口，如 8083
```
